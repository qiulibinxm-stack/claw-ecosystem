- 记录所有决策理由

### 2. 敏捷规划（sprint-prioritizer）

**ICE优先级模型**
- **I**mpact（影响）：预期价值
- **C**onfidence（信心）：成功概率
- **E**ase（难易）：实施难度

**优先级矩阵**
| 优先级 | 特征 | 行动 |
|--------|------|------|
| P0 | 必须做，影响大 | 立即执行 |
| P1 | 应该做，价值高 | 计划执行 |
| P2 | 可以做，有价值 | 视情况执行 |
| P3 | 暂不做，价值低 | 延后或取消 |

**Sprint规划模板**
```markdown
# Sprint #[编号] 规划

## Sprint目标
[一句话描述本次Sprint要达成什么]

## 可用工作日：X天
## 团队容量：X点

## 待办事项
| ID | 事项 | 优先级 | 预估 | 负责人 |
|----|------|--------|------|--------|
| 1 | | P0 | 3点 | |
| 2 | | P1 | 5点 | |

## Sprint承诺
[承诺完成的事项]

## 风险点
⚠️ [识别到的风险]
```

### 6. 工作流自动化（workflow-automator）

**自动化优先级评估**
```
频率高 + 规则明确 + 人工耗时 = 最先自动化
频率低 + 需要判断 + 人工可控 = 最后自动化
```

**自动化工作流模板**
```markdown
# 工作流：[名称]

## 触发条件
- 定时：[cron表达式]
- 事件：[触发事件]

## 执行步骤
1. [步骤1]
2. [步骤2]
3. [步骤3]

## 异常处理
- 失败重试：[策略]
- 告警通知：[方式]
- 人工回退：[流程]

## 监控指标
- 执行成功率：目标>99%
- 平均耗时：目标<[X]分钟
- 失败告警：即时通知
```

**五行自动化矩阵**
| 五行 | 自动化方向 | 工具 |

| 金 | 财务报表自动化 | Excel+BI |

```

│   ├── plans/      # 方案设计
│   ├── meeting/    # 会议纪要

├── templates/      # 模板文件
```

**文档命名规范**
- `YYYY-MM-DD-主题-版本.扩展名`
- 例：`2026-04-24-市场调研方案-v2.md`

**定期清理机制**
- 每月清理过期临时文件
- 每季度归档已完成项目
- 保留所有版本的可追溯性

### 8. 任务规划（任务规划师）

**任务分解方法**
- WBS（工作分解结构）：大目标→子任务→具体动作
- 时间估算：乐观/悲观/最可能，取加权平均
- 依赖关系：前置任务→并行任务→后续任务

**甘特图模板**
```markdown
# 项目甘特图 [项目名]

| 任务 | 负责人 | 开始 | 结束 | 状态 | 依赖 |

| 技术方案 | 程流云 | W2 | W2 | 🔄 | 需求分析 |
```

### 9. BI报表自动化（bi-report-auto）

**报表体系设计**
```markdown
# BI报表框架

## 日报（每日自动化）
| 指标 | 数据源 | 计算方式 |
|------|--------|----------|
| 日活用户 | API | count(distinct user) |
| 新增关注 | API | count(new_follow) |
| 内容发布数 | DB | count(content) |

## 周报（每周一自动生成）
- 环比趋势图
- 异常指标标注
- 周度OKR进度

## 月报（每月1日自动生成）
- 月度KPI达成率
- 同比/环比分析
- 下月目标与计划
```

**自动化实现**
- 数据采集：定时脚本/API调用
- 数据处理：Python pandas/SQL
- 可视化：matplotlib/Excel图表
- 分发：邮件/IM自动推送
