## 🎯 AARRR增长模型

### Acquisition 获取
```
渠道分析矩阵
| 渠道 | 流量 | CAC | 质量 | 优先级 |
|------|------|-----|------|--------|
| 自然搜索 | 高 | ¥0 | ⭐⭐⭐⭐⭐ | P0 |
| 付费广告 | 高 | ¥50 | ⭐⭐⭐⭐ | P1 |
| 社交媒体 | 中 | ¥0 | ⭐⭐⭐ | P2 |
| 推荐 | 低 | ¥20 | ⭐⭐⭐⭐⭐ | P1 |
```

### Activation 激活
```
激活漏斗
访问 → 注册 → 首次使用 → 核心价值
100%   25%    15%        达成✅

优化方向：
1. 降低注册门槛
2. 缩短首次价值达成时间
3. 引导用户完成关键动作
```

### Retention 留存
```
留存曲线分析
Day 1: 60%  ← 关键！第一天激活
Day 7: 35%  ← 第一周召回
Day 30: 20% ← 月活基准
Day 90: 15% ← 忠诚用户

召回策略：
- Push通知
- 邮件序列
- 短信触达
- 活动召回
```

### Revenue 收入
```
转化漏斗
访问 → 感兴趣 → 付费意愿 → 付款
100%   30%      10%         5%

优化方向：
- 定价策略
- 促销设计
- 支付体验
- 升级路径
```

### Referral 推荐
```
病毒循环设计
用户 → 推荐动机 → 分享 → 新用户 → 奖励
              ↓
        再次使用

推荐激励设计原则：
1. 双向奖励（邀请人和被邀请人都有）
2. 门槛适当（不要太容易也不要太难）
3. 实时反馈（立即看到推荐效果）
```

## 📊 增长实验框架

### ICE优先级评分
```python
def ice_score(impact, confidence, ease):
    return impact * confidence * ease

# 示例
ice = ice_score(impact=8, confidence=7, ease=5)
# ICE = 280 → 优先执行
```

### 实验设计模板
```markdown
# 增长实验 #EXP-XXX

**将导致**: [Y结果]
**因为**: [Z理由]

## 成功指标
- 主要指标：[指标名] - 目标 +X%
- 护栏指标：[不能恶化的指标]

## 实验组 vs 对照组
| 维度 | 对照组 | 实验组 |
|------|--------|--------|
| 变量 | [当前] | [改变] |
| 样本 | 50% | 50% |
| 周期 | 7天 | 7天 |

## 最小样本量
[统计计算]

## 决策
- ✅ 显著提升 → 全量发布
- ⚠️ 部分提升 → 优化后继续
- ❌ 无提升/下降 → 放弃
```
