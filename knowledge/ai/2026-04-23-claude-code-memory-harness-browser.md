## 一、Claude Code记忆系统核心启示

### 架构概览
- "4+1"结构：四个记录层（公司统一→个人全局→仓库共享→本地私有）+ AutoDream整理系统
- 加载逻辑：1→4顺序加载，4→1优先级执行（越本地优先级越高）
- 纯文件系统实现，每条记忆独立.md文件

### 值得吸收的3个实践

#### 1. AutoDream机制（定期记忆整理）
- **触发条件**：距上次整理≥24小时 + 期间≥5个工作session
- **四阶段流程**：定向→收集→整合→剪枝
- **关键操作**：相对时间转绝对时间、解决记忆漂移（Memories Drifted）、维持200行上限
- **对我们的启示**：可以建立定期cron任务，自动整理daily notes→MEMORY.md

#### 2. 时间标准化（防止记忆漂移）
- 所有"昨天""上周"等相对时间自动转为绝对时间戳
- **问题**：相对时间久了会漂移，导致记忆失真
- **我们的行动**：daily notes中严格使用ISO日期或具体时间点，避免相对时间

#### 3. 双轨记录（纠正+认可）
- 同时保存纠正（用户说"不对"）和认可（用户说"做得好"）反馈
- **我们的现状**：只记错误/纠正，忽略正面反馈
- **行动**：daily notes中也记录"用户认可的方向"，作为长期偏好

### 不适用的部分
- 三重压缩（我们有OpenClaw LCM自动管理）
- 失败熔断（不适用我们的场景）
- 独立Agent后台记忆（我们是单Agent系统）

## 二、Harness浏览器自动化

### 核心特点
- WebSocket直连Chrome DevTools Protocol，无框架设计
- 自修复机制：实时编辑helpers.py文件
- 100%开源，支持Claude Code/Codex等LLM
- GitHub: 484星标

### 与我们现有工具的对比
| 维度 | browser-use/xbrowser | Harness |
|------|---------------------|---------|
| 安装复杂度 | 已集成 | 需额外Python环境 |
| CDP连接 | 浏览器工具内置 | WebSocket直连 |
| 自修复 | 无 | 有（编辑helpers.py） |
| 成熟度 | 已验证 | 较新 |

### 结论
- **当前不需要切换**，现有skill足够
- **记录备用**，如果遇到现有skill搞不定的复杂浏览器任务时再考虑
