## 2. 技术架构深度分析

### 2.1 核心流水线

```
输入主题
  │
  ▼
┌──────────────────────┐
│ LLM（通义/GPT/DeepSeek/Ollama）│ → 自动撰写视频文案
└──────────┬───────────┘
  │ 文案
  ▼
┌──────────────────────┐
│ 分镜规划              │ → 将文案拆分为多个分镜（每句一镜）
└──────────┬───────────┘
  │ 分镜列表
  ▼
┌──────────────────────┐
│ 并行处理（分镜级）      │
│ ┌─────────┐ ┌─────────┐ │
│ │ 分镜1    │ │ 分镜2    │ │ ... ← 同时进行
│ │ AI配图   │ │ AI配图   │ │
│ │ TTS语音  │ │ TTS语音  │ │
│ └─────────┘ └─────────┘ │
└──────────┬───────────┘
  │ 配图 + 语音
  ▼
┌──────────────────────┐
│ 视频模板合成           │ → HTML模板渲染 → FFmpeg合成
│ （竖屏/横屏/方形）      │
└──────────┬───────────┘
  │
  ▼
  输出：完整短视频
```

### 2.2 架构设计哲学：ComfyUI原子能力可组合

**核心洞察**：Pixelle-Video的本质是一个**ComfyUI工作流编排器**。

- 每个环节（LLM/TTS/配图/合成）都是独立的ComfyUI工作流节点
- 工作流以JSON文件定义，放在`workflows/`目录下
- 系统自动扫描目录，动态加载可用工作流
- 用户可以自由替换/定制任何环节的模型和参数

**架构优势**：
| 优势 | 说明 |
|------|------|
| 组件可替换 | 换生图模型？换TTS引擎？直接替换工作流JSON即可 |
| 风格可定制 | FLUX→ChatTTS，完全自由组合 |
| 流程可视化 | 工作流文件即配置，所见即所得 |
| 生态丰富 | ComfyUI社区有大量现成工作流可复用 |

### 2.3 关键技术选型

| 模块 | 技术选型 | 可替换性 |
|------|----------|----------|
| **Web框架** | Streamlit | 硬编码，不易替换 |
| **LLM** | OpenAI兼容API | ✅ 通义/GPT/DeepSeek/Ollama |
| **配图** | ComfyUI（FLUX/Qwen） | ✅ 任何ComfyUI模型 |
| **TTS** | Edge-TTS / Index-TTS | ✅ ComfyUI工作流可替换 |
| **视频合成** | FFmpeg + HTML模板 | ✅ 自定义HTML模板 |
| **BGM** | 内置/自定义MP3/WAV | ✅ 自由替换 |
| **部署** | Windows一键包 / uv源码 | 两种模式 |

### 2.4 模板体系设计

```
templates/
├── static_*.html   → 静态模板（纯文字样式，无需AI生成媒体）
├── image_*.html    → 图片模板（AI生成图片作为背景，最常用）
└── video_*.html    → 视频模板（AI生成视频作为背景，最炫酷）
```

- 支持竖屏(9:16)/横屏(16:9)/方形(1:1)
- HTML模板 → 渲染分镜 → FFmpeg合成视频
- 这是一种**声明式视频合成**思路，值得借鉴

## 3. 代码质量评估

### 3.1 优点
- **ComfyUI工作流JSON化**：能力原子化，组件可组合，设计优秀
- **Streamlit快速原型**：Web UI零代码，适合MVP验证
- **Windows一键包**：降低用户门槛，商业友好
- **Apache 2.0**：商业使用无限制

### 3.2 不足
- **Streamlit硬编码**：无法替换为其他Web框架，扩展性受限
- **50+ Open Issues**：社区活跃但问题积压
- **版本v0.1.15**：仍未到1.0稳定版，API可能有Breaking Change
- **缺乏批量/调度能力**：无法批量生成/定时生成/队列管理
- **缺乏API接口**：只能通过Web UI操作，无法程序化调用

## 5. 部署可行性评估

### 5.1 本机部署方案

| 方案 | LLM | 配图 | 成本 | 显卡需求 |
|------|-----|------|------|----------|
| **完全免费** | Ollama本地 | ComfyUI本地 | ¥0 | NVIDIA 8GB+ |
| **推荐方案** | 通义千问API | ComfyUI本地 | ≈¥0.3/视频 | NVIDIA 8GB+ |
| **云端方案** | OpenAI | RunningHub | ≈¥3-5/视频 | 无 |

### 5.2 部署步骤（Windows）
1. 下载v0.1.15一键整合包
2. 解压 → 双击`start.bat`
3. 配置LLM API Key（推荐通义千问）
4. 配置ComfyUI地址或RunningHub Key
5. 输入主题 → 生成视频
