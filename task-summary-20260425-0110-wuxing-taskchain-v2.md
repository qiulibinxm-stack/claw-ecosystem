## 用户需求
投喂任务不应是"选了谁就谁做"，而是：
1. 用户选了某个虾 → 那个虾收到任务
2. 虾评估是否擅长 → 不擅长则反馈万能虾
3. 万能虾重新调度 → 分派给合适的虾
4. 合适的虾收到 → 接受 → 执行 → 完成反馈万能虾
5. 万能虾收到结果 → 同步到QClaw

## 调度流程设计

### 流程A：直接执行（选对了虾）
```
📥 收到投喂 → 📤 转发给目标虾 → 🔍 评估 → ✅ 确认承接 → ⚙️ 执行 → ✅ 完成 → 📤 汇报万能虾 → 🦞 确认 → 💬 QClaw同步
```

### 流程B：重调度（选错了虾，50%概率触发）
```
📥 收到投喂 → 📤 转发给目标虾 → 🔍 评估 → ⚠️ 不是我的专长 → 📤 反馈万能虾 → 🦞 重新调度 → 📤 派给新虾 → ✅ 新虾承接 → ⚙️ 执行 → ✅ 完成 → 📤 汇报 → 🦞 确认 → 💬 QClaw同步
```
