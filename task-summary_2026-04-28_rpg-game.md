# 五行修炼塔 - RPG风格游戏设计

## 目标
将简单消除游戏升级为RPG风格的"五行修炼塔"，增加关卡、敌人、血量、灵石等元素，提升趣味性和成就感。

## 核心设计

### 🏯 修炼塔概念
- **10层塔**，每层一个敌人
- 敌人越来越强：小妖(5HP) → 精怪(8HP) → 妖将(12HP) → ... → 混沌(150HP)
- 击败敌人获得灵石奖励，进入下一层

### ❤️ RPG元素
- **生命值(HP)**：3点，步数用尽扣1点，归零则修炼失败
- **灵石(💰)**：击败敌人获得，可累积
- **连击(⚡)**：连续匹配增加伤害倍率
- **总分(🏆)**：全程累计分数

### ⚔️ 战斗机制
- 三连消除 → 对敌人造成伤害
- 伤害 = 匹配数 × (1 + 连击×0.5)
- 连击越高，伤害越大
- 步数限制：每层10+关卡×2步

## 技术实现

### CSS（~120行）
- `.game-tower` — 游戏容器
- `.game-header` — 关卡+血量
- `.game-grid` / `.game-cell` — 5×3网格
- `.game-stats` — 连击/灵石/总分
- `.game-progress` — 步数进度条
- `.game-msg` — 实时提示

### JS（~200行）
- `initGame()` — 初始化关卡和敌人
- `selectCell(idx)` — 选中/交换逻辑
- `findMatches()` — 三连检测
- `checkGameState()` — 胜负判定
- `setMsg(text)` — 提示消息

### 游戏状态
```javascript
game = {
  level: 1,        // 当前层数
  hp: 3,           // 生命值
  gold: 0,         // 灵石
  score: 0,        // 总分
  combo: 0,        // 连击
  enemy: {...},    // 当前敌人
  enemyHp: 5,      // 敌人血量
  moves: 0,        // 已用步数
  targetMoves: 12  // 步数限制
}
```

## 趣味性提升

### 对比原版
| 维度 | 原版消除 | 修炼塔 |
|------|----------|--------|
| 目标感 | ❌ 无目标 | ✅ 10层塔，击败敌人 |
| 进度感 | ❌ 无关卡 | ✅ 层数、步数进度条 |
| 成就感 | ❌ 纯分数 | ✅ 灵石、通关、RPG元素 |
| 紧张感 | ❌ 无压力 | ✅ 步数限制、生命值 |
| 反馈感 | ⚠️ 简单 | ✅ 击败提示、连击伤害 |

### 心理钩子
1. **目标明确**：看到"第N层"，知道进度
2. **即时反馈**：每次消除都有伤害数值
3. **风险压力**：步数有限，生命有限
4. **成长感**：灵石累积，分数上涨
5. **挑战性**：敌人越来越强
